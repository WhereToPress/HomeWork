3. Заходим в акаунт GitHub
4. Создать внешний репозиторий с назанием JSON
- клик "+", New Repository, name - "JSON" scroll down, click button "Create repository" 
5. Клонировать репозиторий на локальный компьютер
- в консоле: git clone https://github.com/WhereToPress/JSON.git
6. Внутри локального JSON создать файл “new.json” 
- в кансоле: cd json. touch new.json
7. Добавить файл под гит 
- в консоле: git status. git add new.json
8. Закоммитить файл 
- в консоле: git commit -m "Add file 'new.json'"
9. Отправить файл на внешний GitHub репозиторий. 
- в консоле: git push
10. Отредактировать содержание файла “new.json”: написать информацию о себе (ФИО, возраст, количество домашних животных, будущая желаемая зарплата). Всё написать в формате JSON.
- в консоле: nano new.json. Write text. Ctrl+O. Enter. Ctrl+X
11. Отправить изменения на внешний репозиторий.
- в консоле: git status. git add new.json. git commit -m "Changed file new.json". git push
12. Создать файл preferences.json
- в консоле: touch preferences.json
13. В файл preferences.json добавить информацию о своих предпочтениях (Любимый фильм, любимый сериал, любимая еда, любимое время года, сторона которую хотели бы посетить) в формате JSON.
- в консоле: nano preferences.json. Write text. Ctrl+O. Enter. Ctrl+X
14. Создать файл sklls.json добавить информацию о скиллах которые будут изучены на курсе в формате JSON
- в консоле: touch sklls.json. nano sklls.json. Write text. Ctrl+O. Enter. Ctrl+X
15. Отправить сразу 2 файла на внешний репозиторий.
- в консоле: git status, git add ., git commit -m "Add files: preferences.json sklls.json", git push
16. На веб интерфейсе создать файл bug_report.json.
- в GitHub.com, в репозитории "JSON", клик "Add file", "Create new file" bug_report.json
17. Сделать Commit changes (сохранить) изменения на веб интерфейсе.
- в GitHub.com, скроллим вниз, заполняем поле "Commit changes", click button "Commit changes"
18. На веб интерфейсе модифицировать файл bug_report.json, добавить баг репорт в формате JSON.
- в GitHub.com, в репозитории "JSON" клик по файлу "bug_report.json", Edit this file, Write text
19. Сделать Commit changes (сохранить) изменения на веб интерфейсе.
- в GitHub.com, скроллим вниз, заполняем "Commit changes", click button "Commit changes"
20. Синхронизировать внешний и локальный репозиторий JSON
- в консоле: git pull
